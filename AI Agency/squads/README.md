# Squads

Cada squad representa um pipeline reutilizável para um tipo de serviço.

## Estrutura recomendada

- `squad.yaml` com identidade do squad
- `pipeline.yaml` com passos e agentes
- `memory.md` com aprendizagem útil do squad
- `checklists/` com critérios de QA
- `templates/` com briefs e artefactos
- `runs/` com execuções concretas
